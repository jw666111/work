# AI 平台 Coding Plan 套餐包选购页面 - 分析文档

## 🔬 阶段 1: 用户研究专家

### 目标用户画像

#### 主要用户群 - 独立开发者
- **角色**：个人开发者、自由职业者、Side Project 爱好者
- **年龄段**：22-35 岁
- **技术水平**：中级到专家
- **使用场景**：日常编码、学习新技术、快速原型开发
- **核心诉求**：性价比高、按需付费、不想为用不到的功能买单

#### 次要用户群 - 小团队 Tech Lead
- **角色**：初创公司 CTO、技术负责人
- **使用场景**：给团队采购、统一管理额度
- **核心诉求**：团队共享额度、用量统计、成本可控

#### 边缘用户 - 尝鲜用户
- **角色**：听说 AI 编程很火，想试试
- **核心诉求**：免费试用、门槛低、快速体验价值

### 用户痛点分析

| 痛点 | 严重程度 | 频率 | 现有方案的问题 |
|------|----------|------|----------------|
| 套餐太贵，用不完浪费 | 高 | 高 | 只有月付 $20，轻度用户心疼 |
| 不知道哪个模型适合我 | 高 | 中 | 没有对比说明，全靠猜 |
| 额度用完了突然断掉 | 中 | 中 | 没有提醒，体验割裂 |
| 团队想共享但各自付费 | 中 | 低 | 没有团队版 |
| 账单看不懂 | 中 | 中 | 只显示总价，不知道钱花哪了 |

### 竞品分析

| 产品 | 套餐 | 优点 | 缺点 | 可借鉴 |
|------|------|------|------|--------|
| **Cursor** | Free / Pro $20 / Business $40 | 简单粗暴，不用选 | 中间档缺失，$20 对轻度用户贵 | 定价锚点清晰 |
| **GitHub Copilot** | Individual $10 / Business $19 | $10 门槛低，接受度高 | 功能单一，只是补全 | $10 是心理舒适区 |
| **Replit AI** | 按 Cycles 计费 | 用多少付多少 | 计费复杂，算不清楚 | 灵活性思路 |
| **Claude/ChatGPT** | Free / Plus $20 / Pro $200 | 高端选项满足重度用户 | $20 到 $200 之间断层 | Pro 级高毛利定位 |

### 用户旅程地图

```
[听说AI编程] → [注册试用] → [体验小额度] → [额度用完] → [考虑付费] → [选套餐] → [付款] → [持续使用]
     😮          😊          🤩           😰         🤔         😵       💸      😊/😤
    好奇         期待          惊艳          焦虑        纠结     选择困难   心疼/值得  满意/后悔
```

**关键转化点**：额度用完 → 付费 这一步流失最多

### 设计建议

#### 必须做 (Must Have)
1. 提供 $10 以下的入门档 - 降低付费门槛
2. 清晰的套餐对比表 - 减少选择困难
3. 额度不足提醒 - 避免体验断裂

#### 应该做 (Should Have)
1. 按量付费选项 - 满足轻度/波动用户
2. 模型能力说明 - 帮用户选对
3. 用量预估工具 - "像你这样的用户通常选 xx"

#### 可以做 (Nice to Have)
1. 年付折扣 - 提高 LTV
2. 团队版 - 拓展 B 端
3. 学生优惠 - 培养未来用户

## 📋 阶段 2: PM 需求分析

### 背景与目标

基于用户研究，我们需要设计一个 Coding Plan 套餐选购页面，目标：

- **提高付费转化率**：降低门槛，让更多免费用户升级
- **提高 ARPU**：通过合理分档，让用户选择适合的高价值套餐
- **减少选择困难**：清晰对比，快速决策

### 套餐设计

| 套餐 | 价格 | 定位 | 核心卖点 |
|------|------|------|----------|
| **体验** | 9.9 | 体验 | 限额 1 次，体验 AI 编程 |
| **月付包** | 99/月 | 主力   | 2000 次/5 小时 + GPT-4/Claude，专业开发者首选 |
| **加量包** | 40/次  | 补充包 | 定额                                          |



### 功能清单

#### P0 - 核心功能
- [x] 套餐卡片展示（3档）

- [x] 功能对比表

- [x] 当前套餐标识

- [x] 支付流程（Stripe）

- [x] 使用明细

  

#### P1 - 重要功能
- [ ] 模型能力对比弹窗
- [ ] 用量进度条 + 剩余额度
- [ ] "推荐"标签（Pro 档）
- [ ] 套餐变更预览（升级立即生效/降级下月生效）

#### P2 - 增强功能
- [ ] 用量预估计算器
- [ ] 月付/年付切换（年付 8 折）
- [ ] 学生认证入口
- [ ] 邀请返利

### 验收标准

1. 用户能在 10 秒内理解 3个套餐的区别
2. 从点击"购买"到支付完成不超过 3 步
3. 当前套餐和用量一目了然
4. 支持查看所有调用明细

## 🎨 阶段 3: 设计师出方案

### 交互流程

```
[进入页面]
     ↓
[查看套餐卡片] ←→ [切换月付/年付]
     ↓
[点击"购买"]
     ↓
[跳转 Stripe Checkout]
     ↓
[支付成功 → 返回 + Toast 提示]
```

### 状态定义

| 状态 | 条件 | 视觉表现 |
|------|------|----------|
| 当前套餐 | 用户已购买 | 边框高亮 + "当前" 标签 |
| 已失效 | 用完/过有效期 | list 收起 |
| 未消耗 | 已购买但未使用 | 更多套餐收起 |
| 推荐 | Pro 套餐 | ✨ 标签 + 视觉突出 |

## 📊 核心洞察总结

- 🎯 需要 **入门套餐** + **清晰对比** + **用量提醒**
- 💡 关键转化点：额度用完 → 付费（流失最多）
- ✅ $10 是心理舒适区，可借鉴 GitHub Copilot 定价策略
